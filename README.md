# EduCheck — Система автоматизированной проверки студенческих работ

**EduCheck** — это платформа для преподавателей, предназначенная для автоматизации сбора, анализа и рецензирования программного кода. Система минимизирует рутинную нагрузку на преподавателя, используя статический анализ кода и нейросетевые модели (LLM) для предоставления обратной связи студентам.

## 🚀 Основные возможности

- **Управление учебным процессом**: Создание дисциплин, управление группами и дедлайнами заданий.
- **Автоматизированный сбор**: Интеграция с почтовыми серверами (IMAP) для автоматического получения работ.
- **Интеллектуальное рецензирование**: 
    - Статический анализ кода с использованием **Roslyn**.
    - Автоматический Code Review с помощью ИИ (**Ollama / DeepSeek**).
- **Мониторинг задолженностей**: Автоматическое формирование списков должников по группам и заданиям.
- **Хранение данных**: Надежное хранение исходного кода в S3-совместимом хранилище (**Minio**).

## 🛠 Технологический стек

- **Framework**: .NET 9 (C#)
- **Frontend**: Blazor Server (Bootstrap 5)
- **Database**: PostgreSQL + Entity Framework Core
- **Messaging**: RabbitMQ + MassTransit (для асинхронной обработки задач)
- **Storage**: Minio (S3)
- **Analysis**: Roslyn API, MediatR, Ollama API
- **Architecture**: Clean Architecture + DDD (Domain Driven Design)

## 🏗 Архитектура проекта

Проект построен по принципам чистой архитектуры:

1.  **EduCheck.Core**: Ядро системы. Содержит доменные модели (Aggregates, Entities), Value Objects и доменные события.
2.  **EduCheck.Application**: Слой бизнес-логики. Здесь расположены интерфейсы, MediatR Handlers, сервисы обработки и DTO.
3.  **EduCheck.Infrastructure**: Реализация технических деталей. Включает репозитории, работу с БД, интеграцию с Minio, SMTP/IMAP клиенты и логику работы с AI.
4.  **EduCheck.Web**: Пользовательский интерфейс на Blazor Server.
5.  **EduCheck.EmailWorker**: Фоновый воркер (Background Service). Обрабатывает сообщения из очередей RabbitMQ, выполняет тяжелые задачи по анализу кода и взаимодействию с ИИ.

## 🔄 Жизненный цикл проверки работы

1.  **Событие**: Студент отправляет письмо с решением.
2.  **Захват**: Система (через IMAP) скачивает вложение и сохраняет его в Minio.
3.  **Очередь**: Создается событие `SubmissionAttemptAdded`, которое через **MassTransit** отправляет задачу в **RabbitMQ**.
4.  **Обработка**: `EmailWorker` подхватывает задачу, проводит статический анализ (Roslyn) и отправляет код на ревью в **DeepSeek (через Ollama)**.
5.  **Результат**: Преподаватель видит статус, отчет анализатора и ИИ-рецензию в личном кабинете.

## 💻 Установка и запуск

### Требования
- .NET 9 SDK
- PostgreSQL
- RabbitMQ
- Minio
- Ollama (с загруженной моделью `deepseek-r1`)

### Настройка
1. Склонируйте репозиторий.
2. Настройте строки подключения в `appsettings.json` для проектов `Web` и `EmailWorker`.
3. Примените миграции БД:
   ```bash
   dotnet ef database update --project EduCheck.Infrastructure --startup-project EduCheck.Web
   ```
4. Запустите Docker-контейнеры для инфраструктуры:
   ```bash
   docker-compose up -d
   ```
5. Запустите проекты `Web` и `EmailWorker`.
